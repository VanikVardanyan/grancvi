# Grancvi — обзор проекта

Документ для контекста: что это за продукт и как им пользуются. Спека — в `SPEC.md`, технические правила — в `CLAUDE.md`.

## Что это

Telegram-бот для записи к мастеру: стоматолог, парикмахер, барбер, маникюр, бровист, косметолог. Рынок — Армения. Заменяет блокнот и звонки.

Точка входа в продакшене: `@grancviWebBot`. Внутри — TMA (Telegram Mini App) на `app.jampord.am`, бот для уведомлений и подтверждений, FastAPI бэкенд.

## Кто пользуется

- **Мастер** — ведёт приём, управляет расписанием.
- **Клиент** — записывается к мастеру.
- **Салон** — несколько мастеров под одной вывеской, владелец видит общую картину.
- **Админ платформы** — внутренние операции (онбординг, удаление, разбор).

## Флоу мастера

1. **Регистрация.** Мастер заходит в бот по приглашению или сам, открывает TMA, заполняет профиль: имя, специальность, услуги, рабочие часы, длительность слота.
2. **Онбординг.** Заполнил профиль → отметка `onboarded_at`. После этого мастер виден клиентам по своей ссылке/QR.
3. **QR-стикер.** Мастер получает персональную ссылку и QR-код. Клеит на стол, в Instagram, на дверь.
4. **Получение заявок.** Клиент шлёт запрос на слот → мастеру приходит уведомление в Telegram с кнопками: подтвердить, отказать, предложить другое время, посмотреть историю клиента.
5. **Управление расписанием.** Мастер сам добавляет записи (клиент позвонил), переносит, отменяет. Может закрывать дни/часы (blackouts).
6. **Напоминания.** Бот шлёт напоминания клиенту перед визитом и мастеру — короткий список приёмов на день.

## Флоу клиента

1. **Вход.** Клиент сканирует QR или жмёт ссылку → открывается бот → TMA с профилем мастера.
2. **Выбор услуги и слота.** Видит список услуг, длительность, цену. Выбирает свободное время.
3. **Заявка.** Жмёт "записаться", вводит имя/телефон (или подтягивается из Telegram). Заявка уходит мастеру.
4. **Подтверждение.** Мастер подтверждает → клиент получает уведомление, что запись закреплена. Если мастер отказал или предложил другое время — клиент видит это в боте.
5. **Напоминание.** За N часов до визита приходит напоминание.
6. **Отмена.** Клиент может отменить запись через бот.

## Флоу салона

1. **Создание салона.** Через админский флоу. У салона есть список мастеров.
2. **Подключение мастеров.** Мастер привязывается к салону (или регистрируется сразу под салоном).
3. **Запись через салон.** Клиент может зайти на страницу салона → выбрать мастера → дальше как обычный клиентский флоу. Источник записи помечается как "salon".
4. **Видимость.** Салон видит расписание всех своих мастеров.

## Флоу админа

- Просмотр списка салонов и мастеров через `/v1/admin/...` API.
- Удаление мастеров и салонов (hard-delete).
- Управление специальностями (`specialties`).

## Ключевые продуктовые принципы

- **Request-to-book.** Клиент шлёт заявку, мастер подтверждает. Не "клиент сам забил слот". Причина: мастер знает про занятость то, чего бот не знает.
- **Мастер главнее бота.** Мастер всегда может вручную добавить, перенести, отменить — без подтверждений.
- **Простота интерфейса.** На каждом экране максимум несколько кнопок. Слэш-команды — только как фолбэк.
- **Telegram, не отдельное приложение.** Клиенту не нужно ничего ставить.
- **QR-стикер как точка входа.** Физический артефакт у мастера на столе — главный канал привлечения клиентов.

## Сущности (упрощённо)

- `master` — мастер, его профиль, рабочие часы, услуги.
- `client` — клиент с Telegram ID и телефоном.
- `salon` — салон, к которому могут быть привязаны мастера.
- `service` — услуга мастера: название, длительность, цена.
- `appointment` — запись: мастер, клиент, услуга, время, статус (`pending`, `confirmed`, `cancelled`, `completed`, `rejected`).
- `master_blackout` — закрытое время у мастера (отпуск, перерыв).
- `reminder` — отложенные уведомления через APScheduler.
- `invite` — пригласительный код для онбординга.

## Технические опоры

- **Время.** В БД всё UTC. На границах — конвертация в `Asia/Yerevan`.
- **Race на бронировании.** Unique partial index на `(master_id, start_at) WHERE status IN ('pending','confirmed')`. При коллизии — `IntegrityError` → `SlotAlreadyTaken` → клиент видит обновлённую сетку.
- **FSM-storage.** Redis, не in-memory.
- **i18n.** Русский и армянский, через Fluent-файлы.

## Что в продукте есть на сегодня

- Регистрация и онбординг мастера.
- Профиль мастера, услуги, рабочие часы.
- Клиентский флоу записи через TMA.
- Подтверждение/отказ через Telegram-кнопки.
- Напоминания клиенту и мастеру.
- Multi-master (несколько мастеров на одного владельца).
- Салоны и привязка мастеров к салонам.
- Blackouts (закрытое время).
- Админский API: список и удаление салонов и мастеров.
- Специальности как отдельная таблица.

## Чего пока нет

- Платежи и оплата онлайн.
- СМС-уведомления.
- Веб-дашборд вне Telegram.
- Отмена записи мастером с UI (логика в `BookingService.cancel` есть, кнопок нет).
- Аналитика для мастера (no-show rate, повторные клиенты).
